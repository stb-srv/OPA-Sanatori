## Voraussetzungen

**Linux Server (Produktion):**
- Ubuntu 22.04 / 24.04, Debian 12 oder Rocky Linux 9
- Root-Zugang (für `install-ubuntu.sh`)
- Offene Ports: 80, 443 (nginx), optional 5000

**Lokal (Entwicklung):**
- Node.js ≥ 18 ([nodejs.org](https://nodejs.org))
- npm ≥ 9

## 🚀 Linux Server Setup (Empfohlen)

Dies ist der empfohlene Weg für den Produktivbetrieb. Das Skript erledigt alles vollautomatisch.

```bash
# 1. Repository klonen
git clone https://github.com/stb-srv/OPA-Santorini.git /opt/opa-santorini
cd /opt/opa-santorini

# 2. Installer starten
chmod +x install-ubuntu.sh
sudo ./install-ubuntu.sh
```

Der Installer richtet interaktiv folgendes ein:

| Schritt | Was passiert |
|---|---|
| System-Update | `apt upgrade`, Basis-Pakete, `openssl` |
| Node.js 20 LTS | Installation via NodeSource |
| PM2 | Prozessmanager mit Autostart nach Reboot |
| `.env` | **Wird automatisch erstellt** – PORT, CORS & ADMIN_SECRET werden direkt befüllt |
| nginx | Reverse Proxy für Port 80 (optional) |
| Firewall (UFW) | Port 80/443 freigegeben, 5000 nur lokal |

Nach dem Installer läuft das CMS sofort unter `http://<deine-domain>/admin`.
Der **Setup-Wizard** startet automatisch beim ersten Aufruf.

> 💡 **SMTP danach konfigurieren** (für Reservierungs-E-Mails):
> ```bash
> nano /opt/opa-santorini/.env
> pm2 restart opa-cms
> ```

## ⚙️ Konfiguration (.env)

Die `.env` wird beim Installer **automatisch erstellt**. Bei manueller Installation:

```bash
cp .env.example .env
nano .env
```

```env
# Server
PORT=5000                     # Port des CMS
ADMIN_SECRET=<auto-generated> # JWT-Secret – wird automatisch generiert, NICHT ändern!
LICENSE_SERVER_URL=https://licens-prod.stb-srv.de
DEV_MODE=false

# SMTP – für Reservierungsbestätigungen per E-Mail
SMTP_HOST=smtp.dein-provider.de
SMTP_PORT=465
SMTP_SECURE=true              # true = SSL/TLS | false = STARTTLS
SMTP_USER=ihre-email@restaurant.de
SMTP_PASS=ihr-smtp-passwort
SMTP_FROM="OPA! Santorini" <noreply@restaurant.de>

# CORS – Alle Domains die auf den Server zugreifen (kommagetrennt)
# Beispiel: https://meinrestaurant.de,https://www.meinrestaurant.de
CORS_ORIGINS=https://meinrestaurant.de
```

## 🔑 ADMIN\_SECRET – automatisch generiert

Der `ADMIN_SECRET` signiert alle Login-Tokens (JWT). Er wird beim Installer **automatisch als sicherer 64-stelliger Hex-Wert generiert** und direkt in die `.env` geschrieben:

```bash
# Was der Installer intern macht:
openssl rand -hex 32
# Ergebnis z.B.: a3f8c2d1e9b047...  (64 Zeichen)
```

Du musst diesen Wert **nicht manuell setzen**. Wichtige Hinweise:

- **Nicht ändern** nach dem ersten Login – sonst werden alle bestehenden Sessions ungültig
- **Nicht committen** – `.env` ist in `.gitignore` eingetragen
- Bei manueller Installation ohne Skript: Wert selbst generieren mit `openssl rand -hex 32`

## 🔄 System aktualisieren

### One-Liner (Linux, empfohlen)

```bash
cd /opt/opa-santorini && git pull && npm install --silent && pm2 restart opa-cms
```

Oder mit dem mitgelieferten Skript:

```bash
./update-mac-linux.sh
```

### Was beim Update passiert

1. Neuen Code von GitHub pullen (`git pull`)
2. Neue/geänderte npm-Pakete installieren
3. CMS-Prozess in PM2 neu starten
4. `.env` und Daten bleiben **unangetastet**

> ⚠️ Bei größeren Updates im Changelog prüfen ob `.env`-Variablen hinzugekommen sind – ggf. `.env.example` mit der eigenen `.env` abgleichen.

## 🔐 Admin-Passwort zurücksetzen

Falls der Zugang zum CMS verloren geht:

```bash
cd /opt/opa-santorini
node reset-admin.js
pm2 restart opa-cms
```

Das Skript setzt das Admin-Passwort zurück und gibt die neuen Zugangsdaten direkt in der Konsole aus.

## 💻 Lokale Entwicklung (Windows / Mac)

Für Tests und Entwicklung auf dem eigenen Rechner:

**Windows:** Doppelklick auf `start-windows.bat`

**Mac / Linux:**
```bash
chmod +x start-mac-linux.sh
./start-mac-linux.sh
```

Beide Skripte erstellen automatisch eine `.env` aus `.env.example` falls noch keine vorhanden ist, und weisen interaktiv auf die nötigen Anpassungen hin.

## ✨ Features

### 🍽️ Speisekarten-Verwaltung
- Gerichte mit Bild, Preis, Nummer & Beschreibung
- Kategorien, Allergene & Zusatzstoffe
- PDF-Export der Speisekarte
- JSON-Backup & Restore (Import/Export)
- Plan-Limit-Prüfung beim Speichern

### 📅 Reservierungen
- Online-Buchung mit Echtzeit-Verfügbarkeitsprüfung
- Tisch-Zuweisung & Kombinationstisch-Logik
- Bestätigungs-/Storno-Links per E-Mail
- Warteliste / Anfrage-Modus wenn voll
- Konfigurierbarer Puffer & Aufenthaltsdauer

### 🌐 Website-Editor
- Startseite, Hero-Bereich, Galerie & Öffnungszeiten
- Impressum, Datenschutz & Cookie-Banner
- Standort-Karte einbetten
- Urlaubs- & Feiertagsverwaltung

### 🔑 Lizenz-System
- FREE / STARTER / PRO / ENTERPRISE Pläne
- 30-Tage-Trial beim Setup
- Plan-Module einzeln manuell überschreibbar (Admin)
- Validierung gegen externen Lizenzserver

### 🔌 Plugin-System
- Erweiterungen mit eigenem `server.js` & Frontend
- Aktivierung/Deaktivierung per Toggle im CMS

## 🛠️ Tech Stack

| Bereich | Technologie |
|---|---|
| Backend | Node.js, Express |
| Datenbank | JSON-Flat-File via `database.js` |
| Auth | JWT + bcrypt |

## 📁 Projektstruktur

```

├── server.js              # Express-Server & alle API-Routen
├── config.js              # Konfiguration (Port, Secrets, SMTP)

├── cms/                   # Admin-Interface
│   ├── index.html
│   ├── app.js
│   ├── style.css
```
